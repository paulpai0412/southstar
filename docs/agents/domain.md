# Domain docs

Southstar is a single-context repository. Before exploring or modifying the codebase, read:

- `CONTEXT.md` for current architecture vocabulary and durable boundaries.
- Relevant decisions under `docs/adr/`.
- `AGENTS.md` for the active implementation contract and verification rules.

Use terms as defined in `CONTEXT.md`. If a required concept is absent, reconsider whether it belongs to the existing vocabulary or record the gap for `domain-modeling`; do not silently invent synonyms.

Surface conflicts with an active ADR explicitly instead of overriding it. Missing optional context files are not blockers.
